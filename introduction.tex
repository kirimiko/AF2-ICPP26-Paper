\section{Introduction}

Protein structure prediction is a long-standing challenge in computational biology due to its fundamental importance in understanding biological functions and developing therapeutics~\cite{anfinsen1973principles}. Traditional approaches, including homology modeling and physics-based simulations, have achieved partial success but often suffer from limited accuracy or extremely high computational cost. The release of AlphaFold2~\cite{jumper2021highly,evans2021protein} by DeepMind represents a major breakthrough, achieving near-experimental accuracy in predicting three-dimensional protein structures from amino acid sequences. Despite its success, the practical deployment of AlphaFold2 remains limited by its high computational cost, as a single inference requires substantial hardware resources.

To accelerate deep learning workloads such as AlphaFold2, modern processors increasingly integrate specialized matrix units that efficiently execute dense linear algebra operations. These units significantly improve throughput by supporting high-performance matrix multiplication and reduced-precision computation such as FP16. Most existing optimizations for AlphaFold2, on both CPUs and GPUs, rely on matrix-multiply-accumulate(MMA)-oriented execution models and highly optimized GEMM kernels.

However, an emerging class of processors adopts a different design by introducing dedicated outer-product-based matrix units, such as Arm Scalable Matrix Extension (SME). Unlike traditional MMA-based architectures, outer-product units organize computation around fine-grained outer-product accumulation with explicit control over operand reuse and accumulation order. Although mathematically equivalent, this execution model leads to different instruction granularities, register usage, and optimal tiling strategies. Consequently, existing AlphaFold2 optimizations designed for MMA-based architectures cannot be directly applied to such processors. As ARM CPU is increasingly adopted in HPC systems (e.g., Fugaku) and stream processors (e.g., M4, 72F), leveraging acceleration units in ARM CPU is critical for performance.
This architectural shift motivates revisiting AlphaFold2 optimization for CPUs with outer-product computation units. In this work, we target the 72F processor, a many-core ARM CPU with eight NUMA domains and per-core SME units. The processor also provides high-bandwidth on-package memory and native FP16 support, making it a promising platform for accelerating large-scale scientific models.

Optimizing AlphaFold2 on this architecture presents several challenges. First, AlphaFold2 is dominated by dense linear algebra operations, requiring highly efficient GEMM kernels that fully exploit the outer-product matrix units. Moreover, our analysis reveals that a large fraction of the GEMM operations in AlphaFold2 are \textit{tall-and-skinny} matrices, which are difficult to optimize using conventional implementations. Second, reduced-precision inference on CPUs introduces additional complexity, as mixed-precision computation must be carefully integrated without degrading prediction accuracy. Third, the many-core multi-NUMA architecture requires efficient communication and parallelization strategies to avoid synchronization overhead and inter-node bandwidth contention. Finally, the variable protein sequence length makes workload partitioning challenging, as overly aggressive parallelism on short sequences may lead to inefficient execution.

To address these challenges, we present MatrixFold, a highly optimized mixed-precision implementation of AlphaFold2 designed for CPUs with outer-product matrix units. MatrixFold includes a tailored GEMM implementation that efficiently maps AlphaFold2 workloads to SME units, together with mixed-precision inference to reduce computational cost. We further design NUMA-aware communication and adaptive parallelism strategies to efficiently utilize the many-core architecture.

Our contributions are summarized as follows:

\begin{itemize}
    \item We present MatrixFold, the first mixed-precision implementation of AlphaFold2 inference on a many-core CPU architecture with outer-product matrix units.
    \item We design outer-product-accumulator-based GEMM kernels tailored to the 72F processor, effectively addressing the prevalence of \textit{tall-and-skinny} GEMM operations in AlphaFold2.
    \item We propose adaptive parallelism and NUMA-aware communication strategies to improve scalability on many-core multi-NUMA systems.
    \item We conduct a comprehensive evaluation on the CASP14 dataset, showing that MatrixFold achieves 3.4$\times$ speedup at maximum over the state-of-the-art GPU implementation FastFold on A800 for long sequences while maintaining prediction accuracy.
\end{itemize}
